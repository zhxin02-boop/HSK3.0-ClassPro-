# Chinese ClassPro（HSK3.0）最小产品骨架确认稿

> 本稿用于本周确认“轻量版产品骨架”。目标不是一次性定义完整平台，而是锁定 HSK1 样课闭环所需的最小可用产品形态。

## 一、本周产品判断

本周采用“轻骨架 + HSK1 样课先行”策略：

1. 先课堂，后平台。
2. 先样课，后批量。
3. 先数据闭环，后 AI 豪华功能。
4. 先能上课，后做成系统。

第一版产品不追求“完整 HSK1-5 系统”，只追求一节 HSK1 样课能真实经历：

`学生预习提交 -> 教师课前看懂 -> 课堂全员互动 -> 教师批改/反馈 -> 学生课后查看反馈`

## 二、最小产品一句话定义

Chinese ClassPro（HSK3.0）轻量版，是一个基于 HTML/GitHub Pages + GAS + Google Sheets 的国际中文 HSK 教学闭环工具，帮助教师把一节课从课前预习、课中互动到课后反馈的数据跑通。

## 三、第一版用户角色

| 角色 | 第一版要完成的事情 | 暂不做 |
|---|---|---|
| 学生 | 打开链接，完成 15 分钟以内预习/测试，课堂参与互动，下课后查看反馈 | 注册登录、个人主页大系统、长期成长社区 |
| 教师 | 查看提交数据，课前判断薄弱点，课堂发起互动，课后批改开放题 | 多教师协作、复杂权限、跨班级管理 |
| 系统 | 记录数据，汇总数据，展示反馈 | 自动生成全套课程、复杂 AI 诊断 |

## 四、第一版产品模块

### 命名边界

为避免学生端和教师端混淆，第一版命名遵守：

| 名称 | 使用场景 | 说明 |
|---|---|---|
| ClassReady+ | 课前预习、课前测试、提交与反馈查询 | 学生课前打开，数据进入 Sheets，教师可批改 |
| ClassPro+ | 课中互动、教师发题、学生实时参与、PK/排行榜 | 课堂上使用，强调全班互动 |
| Teacher Dashboard / Review | 教师看板、批改、学情查看 | 教师课前/课后使用 |

因此，`source/pre-class/HSK1-L02_课堂测试.html` 应命名为 ClassReady 课前测试，不再称为 ClassPro 测试。

### 1. 学生端：ClassReady+

最小功能：

- 进入 HSK1 样课页面。
- 输入姓名。
- 完成微步骤预习/课堂测试。
- 客观题自动记录得分。
- 开放题提交给教师批改。
- 课后按姓名查询教师反馈。

第一版控制：

- 总时长不超过 15 分钟。
- HSK1 题型低压力、短任务、强支架。
- 优先使用图片、拼音、英文解释、点击选择，减少键盘输入。

### 2. 教师端：Teacher Review / Dashboard

最小功能：

- 查看每个学生是否提交。
- 查看每个模块得分、总分、薄弱点。
- 查看开放题原文。
- 写入教师批语和分数。
- 让学生端能查到批改结果。

第一版看板只回答三个问题：

1. 谁做了？
2. 哪里错得多？
3. 我上课前要重点讲什么？

### 3. 课中端：ClassPro+

最小功能：

- 教师端能打开 HSK1 样课课中原型。
- 至少包含 3 类可上课互动：
  - 词汇识别/选词。
  - 拼音或汉字匹配。
  - 句子排序/情景任务。
- 课堂能让全班通过手机参与一次真实互动。
- 课堂结果能被教师看到，哪怕第一版只做轻量记录。

第一版不追求复杂游戏池，保留一个可用的 PK/排行榜或全班即时反馈即可。

### 4. 数据层：GAS + Google Sheets

第一版采用轻量云端数据路线：

- 学生端：GitHub Pages 或本地 HTML。
- 数据接口：Google Apps Script。
- 数据存储：Google Sheets。
- 教师批改：独立 HTML 页面或 GAS 内置页面。

最小数据字段：

| 字段 | 用途 |
|---|---|
| timestamp | 记录提交时间 |
| studentName | 区分学生 |
| lesson | 区分课程，如 HSK1-L02 |
| stage | 课前/课中/课后 |
| module | 题型或任务模块 |
| score | 得分 |
| total | 总分 |
| percentage | 正确率 |
| weakPoints | 薄弱点标签 |
| answer | 学生作答 |
| correction | 教师反馈 |
| reviewStatus | 待批改/已批改/无需批改 |

## 五、HSK1 第一版分级原则

HSK1 样课必须遵守：

- 兴趣导向：入门阶段首先要让学生对汉语产生关注、期待和热情。
- 去键盘化：尽量点选、匹配、拖拽。
- 重拼音、汉字、语音感知。
- 强图片、拼音、英文支架。
- 每个任务短、直观、低压力。
- 预习不是“读材料”，而是“先试错，让教师看见盲点”。

HSK1 专属规划：

- 增加“拼音学练”模块，作为 HSK1 入门阶段的专属能力模块。
- 拼音模块重点服务声母、韵母、声调、拼读、音形匹配和听辨。
- 拼音模块不阻塞本周 HSK1-L02 闭环，建议在 HSK1-L02 跑通后开发。

## 六、模块化升级规范

为后期长期升级，ClassPro 每个题型/活动模块应逐步独立成代码文件，避免所有逻辑堆在一个页面里。第一版原则是“能拆则拆，先稳接口，再慢慢重构”。

### 1. 推荐目录结构

```text
source/in-class/
  teacher.html
  student.html
  data_L02.js
  core/
    classpro-runtime.js
    event-bus.js
    scoring.js
  modules/
    vocab-choice.js
    image-guess.js
    pinyin-match.js
    hanzi-match.js
    sentence-ordering.js
    mission-card.js
    pk-quiz.js
  styles/
    classpro-base.css
    modules.css
```

### 2. 每个模块只负责一件事

每个题型模块建议遵守统一接口：

| 接口 | 含义 |
|---|---|
| `moduleId` | 模块唯一名称，如 `pinyin_match` |
| `render(container, question, context)` | 渲染题目 |
| `validate(answer, question)` | 判断答案或生成待批改记录 |
| `getResult()` | 返回学生作答、得分、耗时、薄弱点 |
| `destroy()` | 清理计时器、事件监听、临时状态 |

这样后期如果只升级拼音匹配、PK 或句子排序，不需要改整个 ClassPro。

### 3. 数据和表现分离

- 课程内容放在 `data_L02.js` 或后续 JSON 中。
- 题型逻辑放在 `modules/`。
- 通用计分、提交、MQTT、排行榜放在 `core/`。
- 页面只负责组装，不直接写大量题型细节。

### 4. 旧 ClassPro 代码复用原则

能复用的旧题型优先复用，但要做两件事：

1. 外层补统一接口，保证它能被 `teacher.html` 或后续 runtime 调用。
2. 内部暂时不大改，避免为了“漂亮重构”拖慢样课闭环。

换句话说：第一阶段可以“旧发动机继续跑”，但方向盘和接口要先统一。

## 七、本周暂缓范围

以下内容不进入本周第一版：

- 原生 App。
- 复杂口语 Agent。
- 高级声学诊断。
- 大规模自动生成所有课程。
- 复杂动态游戏池。
- 完整 HSK1-5 批量开发。
- 复杂账号系统和权限系统。

## 八、样课选择确认

本周样课确认继续使用 `HSK1-L02`，理由：

- 已有材料最多。
- 已有预习页面、测试提交页、GAS 数据接口、教师批改页。
- 已有课中数据样例和教师端原型。
- 更适合快速跑通闭环，降低从零迁移风险。

如果后续从教学顺序上必须从 `HSK1-L01` 开始，建议先用 `HSK1-L02` 跑通结构，再把结构迁移到 `HSK1-L01`。

## 九、第一版验收标准

HSK1 样课第一版只要守住六件事，即视为闭环成立：

1. 学生愿意打开。
2. 预习不超过 15 分钟。
3. 数据能进入 Google Sheets。
4. 老师课前能看懂。
5. 课中能让全班动起来。
6. 下课后学生能看到反馈。
