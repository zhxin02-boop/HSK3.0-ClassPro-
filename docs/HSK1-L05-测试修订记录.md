- 课前预习不设置课文细节、人物关系、课文事实判断或根据课文回答的题目。
- 课文相关理解题放在课中输入之后或课后复习中处理。
- 课前只保留主题导入、词汇识别、拼音/词汇预热、语法预热和生活经验主观输出。
- 如果课前题目使用新增例句、练习句或情景表达，必须标记为“教学扩展”。

## 后续课程必须避免的同类问题

### 1. 课前预习

- 课前预习不得考课文细节、人物关系、课文事实判断或课文问答。
- 课前可以出现与主题相关的词汇、拼音、生活经验表达和语法预热，但不能要求学生在学课文前完成课文理解。
- 如果课前使用新增例句或情景句，必须标记为“教学扩展”。

### 2. 词句匹配

- 词句匹配必须是“问句和回答”的匹配，不是普通词语和句子的机械匹配。
- 每组保持4个问句和4个对应回答。
- 回答必须语义唯一，不能出现一个回答可以匹配多个问句的情况。
- 干扰项不能只靠重复关键词制造表面混淆。例如问句和答句都有“做饭”时，仍要通过肯定/否定、具体回答、开放问答等方式保证答案唯一。
- 题目质量检查时要逐组人工判断：学生是否只凭关键词就能完成。如果可以，需要重写。

### 3. 句子转换

- 题目必须包含一个完整原句、一个清晰英文转换要求和四个完整句子选项。
- 选项不能是短语，不能把题干本身写成答案。
- 题目本身必须有转换意义；避免“只是删掉/合并一个无意义成分”的机械题。
- 正确答案位置必须打乱，并由数据或通用适配层同步 `correct_index`。
- 拼音必须使用声调符号并显示在汉字上方，如 `wǒ xǐhuan diànnǎo`；不得使用数字声调，如 `wo3 xi3 huan dian4 nao3`。
- 教师端和学生端的拼音呈现必须一致。

### 4. 汉字认读循环

- 教师端和学生端必须使用同一个题目 payload。
- 教师端必须稳定显示“答 / Key / 下一题”，点击“答”后学生端收到当前轮题目，点击“Key”后学生端显示答案。
- 第4轮“短句找词”不能只从同组生词自动抽干扰项；遇到容易多答案的句子时，必须允许课程数据指定干扰项。
- 第5轮“英文阅读理解”也必须允许课程数据指定整句干扰项。
- 干扰项可以来自前几课已学词，不必局限于本课生词。
- 必须检查是否存在多答案，尤其是以下类型：
  - 同义或近义日期词：星期日 / 星期天。
  - 同类食物词：面条儿 / 饺子。
  - 可同时搭配同一句的词：菜 / 一些、下班 / 休息、它 / 喜欢。
  - “我喜欢___”“我会做___”这类开放空格句。
- 第4、5轮不能出现空题、重复题、重复选项或答案文本重复。

### 5. 课后作业

- A/B/C 三档各12题，题目不能重复。
- 尽量避免直接问语法定义或语法术语，优先考交际理解和句子运用。
- 连词成句的词块顺序不能与答案顺序一致。
- 主观题必须有英文提示和可填写答题框，批改链路只显示最后一次提交。

### 6. 通用化原则

- 新课制作时优先改课程数据，不为单课写死页面逻辑。
- 如果某类题型确实需要新能力，先抽象为通用数据字段或通用渲染逻辑。
- 本轮新增的汉字认读干扰项字段应作为后续课程可复用机制：
  - `hanziRecognition.sentenceDistractors`
  - `hanziRecognition.readingDistractors`
