# HSK1-L01 / L02 可部署版本说明

## 当前部署方式

本项目可以作为静态网站部署到 GitHub Pages 或其他静态托管平台。

- 学生入口：`source/student/index.html`
- 教师入口：`source/teacher/index.html`
- 课前：`source/pre-class/index.html?lesson=HSK1-L01`
- 课中：`source/in-class/teacher.html?lesson=HSK1-L01&room=8888`
- 课后：`source/post-class/student-report.html?lesson=HSK1-L01&room=8888`

## 静态部署规则

1. 页面之间使用相对路径，适配 GitHub Pages 的仓库子路径。
2. 课程、房间和学生参数通过 URL 传递，不依赖本地存储才能进入正确课程。
3. MQTT 课堂互动在部署环境直接连接 HiveMQ。
4. 部署环境直接使用 GAS 写入和读取学习记录；本地 `server.js` 仅用于本地测试和本地报告兜底。
5. GitHub Pages 发布后，教师端生成的二维码必须指向发布后的 `source/in-class/student.html`，不能继续使用 localhost。

## 发布前检查

- [ ] GitHub Pages 的发布目录包含 `index.html`、`source/`、`data-model/` 和 `references/`。
- [ ] 用手机访问学生入口，并确认课程参数保留为 `HSK1-L01` 或 `HSK1-L02`。
- [ ] 教师端选择课程后，检查二维码中的学生地址不是 localhost。
- [ ] 检查 MQTT 连接状态和学生答题回传。
- [ ] 检查 GAS 读写权限和课后报告。
