# HSK1-L02 课中新题型接入策略与记录

## 一、当前判断

本阶段重点不是堆题型数量，而是确认每一种互动是否适合 HSK1 入门学生。

判断标准：

- 是否短：学生能在 10-20 秒内理解任务。
- 是否低压力：先选、判断、排序，少打字、少自由输出。
- 是否有即时反馈：学生做完马上知道对不对。
- 是否能服务课堂节奏：教师能快速切换，不需要复杂说明。
- 是否能复用数据：题目、答案、可接受答案尽量从 `data_L02.js` 读取。

## 二、已接入样课的题型

### 1. 完成句子

数据源：

- `classProQuestions.v4_complete_sentence`

页面入口：

- `练习 -> 完成句子`

互动形式：

- 学生看一个带空格的短句。
- 从 3 个选项中选择。
- 页面即时显示正确/错误。

适合 HSK1 的原因：

- 比自由造句更低压力。
- 能训练句子结构，但不要求学生键盘输入。
- 可作为选词填空前后的过渡题。

### 2. 选词填空

数据源：

- `classProQuestions.v5_vocab_fill`

页面入口：

- `练习 -> 选词填空`

状态：

- 已有题型，继续保留。

互动价值：

- 适合作为词汇理解和语法位置判断的快速反馈。

### 3. 课文判断

数据源：

- `classProQuestions.tt_text_true_false`

页面入口：

- `练习 -> 课文判断`

互动形式：

- 学生读一句关于课文内容的陈述。
- 选择 `对 / True` 或 `不对 / False`。
- 页面即时显示正确/错误。

适合 HSK1 的原因：

- 任务非常明确。
- 能检查课文理解，不要求复述。
- 适合放在课文讲解之后作为轻量互动。

### 4. 连词成句

数据源：

- `classProQuestions.g1_ordering`

页面入口：

- `练习 -> 连词成句`

状态：

- 已有题型，继续保留。
- 可接受答案通过 `acceptable_answers` 维护。

互动价值：

- 适合训练结构。
- 第一轮只使用 3 个词块或 4 个词块的短句。

### 5. PK 竞赛

数据源：

- `classProQuestions.pk_question`

页面入口：

- `竞赛 -> PK`

状态：

- 已有题型，继续保留。

互动价值：

- 用于课堂收束和情绪提升。
- 题面已增加英文，降低理解负担。

## 三、本轮暂缓接入的题型

### 图片猜词

暂缓原因：

- 词汇配图还在重做规范阶段。
- 如果图片质量不稳定，会干扰词汇学习重点。

建议：

- 等 HSK1-L02 词汇图片按规范重做后再接入。

### 翻牌配对

暂缓原因：

- 互动形式有趣，但需要更完整的状态管理。
- 若做得太快，可能变成视觉游戏而不是语言练习。

建议：

- 第二批接入，适合作为课后或复习型互动。

### 句子匹配 / 阅读选择

暂缓原因：

- 文本量比完成句子和判断题更高。
- HSK1-L02 第一轮样课先不增加阅读负担。

建议：

- 第二批接入，放在学生已经熟悉课文之后。

### 听力系列

暂缓原因：

- 当前音频资源路径和实际文件还需要统一确认。

建议：

- 音频资源稳定后再接入。

### 汉字部件 / 拼音到汉字

暂缓原因：

- 属于 HSK1 专属拼音/汉字学练模块方向。
- 用户已确认可在 HSK1-L2 跑通后开发。

建议：

- 后续独立成模块，不混入本轮样课主页面。

## 四、当前练习区建议顺序

建议课堂中按以下顺序使用：

1. 完成句子：结构轻练。
2. 选词填空：词汇位置判断。
3. 课文判断：课文理解反馈。
4. 连词成句：结构输出。

这个顺序从低压力识别逐步过渡到结构操作，适合 HSK1 入门学生。
