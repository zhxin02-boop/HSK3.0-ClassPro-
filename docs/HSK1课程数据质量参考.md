# HSK1 课程数据质量参考

本文件用于新课数据生成前、生成中和提交前检查。它沉淀 L01-L05 反复测试后暴露出的共性问题，优先用于避免同类问题在后续课程重复出现。

## 1. 总原则

- 优先新增或修正课程数据文件，不为某一课写死页面逻辑。
- 页面继续通过统一课程上下文读取 `lesson`、`room`、`student`。
- 新增例句、练习句、情景对话必须标记为“教学扩展”；教材课文原句必须保留，不得改写。
- 教师端界面不直接显示英文内部字段名，页面标签统一使用中文或中英教学标签。
- 每课完成后必须独立验证学生端、教师端、批改台、数据面板和学生反馈链路。

## 2. 课程上下文与旧课残留

- 页面标题、完成页、学习报告、批改台、数据面板不能残留旧课程标题、旧 warm-up 文案或固定 `HSK1-L02`。
- 数据记录必须写入当前 `lesson` 和 `room`，不能写死课程编号。
- 教师端和学生端发送/接收同一题目 payload，不能由两端各自临时构造不同题目。
- 学生重连后应能收到当前题目或当前轮状态。

## 3. 课前预习

- 课前预习不得考课文细节、人物关系、课文事实判断或课文问答。
- 课前可以做主题导入、词汇识别、拼音/词汇预热、语法预热和生活经验主观输出。
- 课前主观题必须有英文答题提示、可填写输入框、`needsTeacherReview/openEnded` 标记，并能进入教师批改台。
- 答案不唯一时必须设置 `acceptable_answers`。
- 正确选项不能集中在 A；生成后必须检查答案位置分布。

## 4. 拼音规则

- 拼音必须显示在汉字上方，教师端和学生端保持一致。
- 拼音必须使用声调符号，如 `wǒ xǐhuan diànnǎo`，不得使用数字声调，如 `wo3 xi3 huan dian4 nao3`。
- 拼音练习选项只显示拼音，不显示汉字。
- 选项有汉字时，除汉字区辨题外，应提供对应拼音。
- 专名、课文原句、生词、词块和句子选项都要校对拼音，避免漏声调、错声调和拼音缺失。

## 5. 题目与选项通用质量

- 同一道题的选项格式必须一致，不得混用词、短语、句子或中英文格式。
- 不得出现重复选项、同义重复选项或一个题有多个正确答案。
- 干扰项必须有教学意义，不能只是随机无关词。
- 干扰项可以来自前几课已学词，不必局限于本课生词。
- 题干不能直接泄露答案。
- 不同题型必须体现不同练习维度，不能都退化成选词填空。

## 6. 词句匹配

- 词句匹配必须是“问句和回答”的匹配，不是普通词语和句子的机械匹配。
- 每组保持 4 个问句和 4 个对应回答。
- 回答必须语义唯一，不能一个回答能匹配多个问句。
- 不能只靠关键词重复制造题目。例如问句和答句都有“做饭”时，还要通过肯定/否定、具体回答、开放问答等方式保证答案唯一。
- 左右卡片大小应一致，学生端和教师端显示同一组 pairs。

## 7. 句子转换

- 每题必须包含一个完整原句、一个清晰英文转换要求和四个完整句子选项。
- 选项不能是短语，不能把题干本身写成答案。
- 题目必须有真实转换意义，例如改成疑问句、否定句、替换代词、加入副词、连用“也”等。
- 避免无意义转换，例如只机械合并两个已完整表达的句子。
- 正确答案必须随机打乱，并同步更新 `correct_index`。
- 教师端和学生端必须用同样的题目 payload 与答案索引。

## 8. 汉字区辨

- 汉字区辨不能在题干中直接泄露答案。
- 如果答案是两字词，干扰项最好也保持两字词，避免学生只凭字数判断。
- 干扰项应体现字形或词形区辨，但不能造成多个正确答案。
- 汉字区辨选项默认只显示汉字，不显示拼音。

## 9. 汉字认读循环

- 汉字认读教师端和学生端必须使用完全一致的题目 payload。
- 教师点击“答”后，学生端收到题目；教师点击“Key”后，学生端显示答案；下一题或下一轮必须同步到学生端。
- 第4轮“短句找词”不能只从同组生词自动抽干扰项；遇到多答案风险时，必须通过课程数据指定干扰项。
- 第5轮“英文阅读理解”也必须允许课程数据指定整句干扰项。
- 可复用字段：
  - `hanziRecognition.sentenceDistractors`
  - `hanziRecognition.readingDistractors`
- 必须重点检查以下多答案风险：
  - 同义或近义日期词：星期日 / 星期天。
  - 同类食物词：面条儿 / 饺子。
  - 可同时搭配同一句的词：菜 / 一些、下班 / 休息、它 / 喜欢。
  - 开放空格句：我喜欢___、我会做___、我也___。
- 第4、5轮不能出现空题、重复题、重复选项或答案文本重复。

## 10. 连词成句

- 词块必须带拼音。
- 词块出现顺序不能与答案顺序一致，课后作业尤其要检查。
- `acceptable_answers` 应覆盖合理语序变化。
- 学生端提交后应能给出明确反馈，教师端能记录答题状态。

## 11. 看图猜词与图片资源

- 看图猜词选项必须包含拼音。
- 图片资源可后置，但数据字段应预留图片路径或图片提示。
- 图片不能包含汉字、拼音、英文、水印或字幕，避免泄题。
- 如果图片缺失，应有可接受的文字提示兜底，不影响题目流程。
- 教师端手动上一题/下一题时必须重置当前答题状态，避免侧栏从上一题残留为 Key 或下一题。

## 12. 课后作业

- A/B/C 三档各 12 题，题目不能重复。
- 三档题型和难度要有区分。
- 尽量避免直接问语法定义或语法术语，优先考交际理解和句子运用。
- 主观题必须有英文提示和可填写答题框。
- 最多提交两次；教师批改台只显示最后一次提交；已批改内容自动归档。
- 学生端必须能查看教师反馈。

## 13. 数据记录与批改链路

- 课前主观题、课后输出题和汉字书写都应进入统一待批改工作台。
- 批改台不能依赖固定模块名称判断是否需要批改，应优先使用 `openEnded`、`needsTeacherReview`、`needs_review` 等标记。
- 批改完成后，学生端必须能读取教师反馈。
- 数据面板、课后报告、批改台字段必须统一。

## 14. 生成后必跑检查

- JSON 能正常解析。
- 每种题型题量达标。
- `correct_index` 与 `correct_answer` 一致。
- 正确答案位置分布合理，不能全部为 A。
- 无重复题、重复选项、同义多答案。
- 无旧课程标题、旧 warm-up、旧 lesson id。
- 无英文内部字段直接显示。
- 拼音无数字声调，且位置在汉字上方。
- 主观题有英文提示和批改标记。
- 教师端、学生端、批改台、数据面板和学生反馈链路可独立验证。
