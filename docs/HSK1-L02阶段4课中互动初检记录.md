## 五、已处理风险

### MQTT / CDN 离线保护

问题：

- `teacher.html` 通过 CDN 加载 MQTT 库。
- 如果 `file://` 打开时网络或 CDN 不可用，`mqtt` 变量不存在，可能导致初始化失败。

处理：

- 已在 `_mq()` 中增加离线保护。
- 如果没有 `window.mqtt`，页面进入 `offline` 状态，不影响本地讲授、练习、PK 演示。

### 教师体验反馈与本轮修复

教师体验反馈：

- 词汇模块点击底部箭头时，右侧突然出现互动墙。
- 词汇讲授缺少配图。
- 语法结构下方的语法点释义太简单；HSK1 初学者需要更详细解释。
- 下阶段学生为俄罗斯学生，语法说明需要英、俄双语版本。
- 结构性语法点最好让结构和例句一一对应，便于记忆。
- 第三个语法点应为“也”的用法，不是简单写成 also 的用法。
- 课文模式希望增加“只显示英文翻译”。
- 点击课文模式切换时，计时器会突然出现。
- 学生扫码答题暂未测试。
- 连词成句存在答案不唯一情况，需要教师确认；当前有些答案需要继续校对。
- PK 赛题目需要英文或中英文提示，否则 HSK1 学生看不懂。
- 当前界面基本沿用旧 ClassPro，后续希望做视觉与体验升级。

本轮已处理：

- 将互动墙状态、课文显示模式、计时器状态拆开，避免切换词汇/课文时误触发互动墙或计时器。
- 为已有 8 个核心词汇接入本地配图；暂无图的词汇先保留汉字大卡兜底。
- 语法讲解页接入中文、English、Русский 三语说明。
- 将第三个语法点标题改为“也”的用法，并补充 HSK1 入门解释。
- 课文讲授新增“英文”模式，只显示英文翻译。
- 连词成句检查结果改为“参考答案 + 教师现场确认”语气，并在数据中加入 `acceptable_answers`、`needs_teacher_review` 标记。
- PK 题面在教师端显示中英双语，数据中的英文题面已优化为更自然的课堂英语。
- 已用 Node 检查 `teacher.html` 内联脚本和 `data_L02.js`，语法通过。

仍需确认：

- 学生扫码答题入口和课堂提交链路尚未完成实测。
- 连词成句需要继续做教师审核，尤其是可接受答案列表是否符合 HSK1 教学目标。
- 界面升级需要单独进入 ClassPro+ 轻量 redesign，不建议和本轮功能修复混在一起一次性重做。

## 六、下一步建议

1. 教师先打开 `source/in-class/teacher.html` 做一轮体验确认。
2. 按顺序检查：

   - 课中界面升级优先改哪些区域。
