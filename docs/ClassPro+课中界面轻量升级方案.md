# ClassPro+ 课中界面轻量升级方案

> 本方案服务 HSK1-L02 样课阶段。目标不是重做一个大平台，而是在不拖慢闭环的前提下，让课中端更像 Chinese ClassPro（HSK3.0）的第一版产品。

## 一、升级目标

ClassPro+ 课中界面要解决三个问题：

1. 教师上课时一眼知道现在在哪个环节。
2. HSK1 学生看到屏幕时不紧张、不迷路。
3. 后续题型和拼音模块能逐步独立升级。

第一版关键词：

`清楚、轻、课堂感、低压力、可扩展`

## 二、保留与改变

### 保留

- 当前 `teacher.html` 的基本功能入口。
- 讲授、练习、PK 三大结构。
- 旧 ClassPro 可运行的题型逻辑。
- `data_L02.js` 作为当前样课数据源。

### 改变

- 从“工具面板感”升级为“课堂活动台”。
- 减少同时出现的信息。
- 把互动墙、计时器、排行榜等工具放成可控浮层，不默认打扰主任务。
- 强化 HSK1 的图片、拼音、英文/俄文支架。
- 给后续模块化拆分留下清楚边界。

## 三、界面信息架构

建议采用三层结构：

```text
顶部：课程状态条
中间：课堂主舞台
底部：教学流程控制
侧边：教师工具抽屉
```

### 1. 顶部课程状态条

显示最必要信息：

- Chinese ClassPro+
- HSK1-L02 我叫李文
- 当前环节：词汇 / 语法 / 课文 / 练习 / PK
- 在线状态：离线演示 / 已连接

避免：

- 过多按钮挤在顶部。
- 图标含义不明。
- 连接状态抢主视觉。

### 2. 中间课堂主舞台

主舞台只放当前教学对象：

- 词汇：图片 + 汉字 + 拼音 + 英文。
- 语法：结构公式 + 简明解释 + 例句。
- 课文：当前模式下的文本。
- 练习：题目 + 可操作选项。
- PK：题面 + 队伍得分 + 答案反馈。

原则：

- 一个屏幕只解决一个教学动作。
- HSK1 前几课默认隐藏扩展例句。
- 题目卡片不要层层嵌套。
- 学生远距离看也要清楚。

### 3. 底部教学流程控制

底部只保留：

- 上一步。
- 下一步。
- 当前进度。
- 必要时的检查/重置。

不建议底部放：

- 互动墙开关。
- 复杂设置。
- 与当前任务无关的工具。

### 4. 侧边教师工具抽屉

教师工具建议放在右侧或左侧抽屉：

- 计时器。
- 互动墙。
- 二维码。
- 排行榜。
- 随机点名。

默认收起，教师需要时打开。

这样能避免之前“翻页突然出现互动墙/计时器”的体验问题。

## 四、视觉风格建议

### 1. 气质

ClassPro+ 应该像一块清爽的互动白板，而不是复杂后台。

推荐气质：

- 清楚。
- 友好。
- 有课堂活力。
- 不幼稚。
- 不压迫。

### 2. 色彩

建议使用一套克制但明亮的颜色：

| 用途 | 建议 |
|---|---|
| 主色 | 温和红 / 珊瑚红，用于品牌和重点按钮 |
| 学习蓝 | 用于语法、课文、结构说明 |
| 练习绿 | 用于正确反馈和完成状态 |
| PK 黄/橙 | 用于竞赛，但只在 PK 模块中高亮 |
| 背景 | 浅灰白或暖白 |

避免：

- 大面积深色背景。
- 大面积紫色渐变。
- 过多彩色同时出现。
- 每个模块一套完全不同的视觉语言。

### 3. 字体与字号

教师投屏场景建议：

| 元素 | 建议字号 |
|---|---|
| 汉字主词 | 48-64px |
| 拼音 | 20-28px |
| 英文/俄文支架 | 16-20px |
| 题目 | 24-32px |
| 按钮 | 16-18px |

原则：

- 汉字最大。
- 拼音紧跟汉字。
- 英文/俄文是支架，不抢主视觉。
- 远距离可读优先于信息密度。

## 五、模块级升级建议

### 1. 词汇模块

当前问题：

- 图片质量需要重做。
- 例句对 HSK1 前几课可能认知负荷过高。

升级方案：

- 第一屏只显示：图片、汉字、拼音、英文、朗读按钮。
- 常用短语可保留，但默认不展开或只显示 1-2 个核心组合。
- 扩展例句默认隐藏，由 `meta.showVocabExamples` 控制。
- 图片字段后续放入 `data_L02.js`，页面不再写死图片路径。

### 2. 语法模块

当前已确认：

- 英俄双语说明适合俄罗斯 HSK1 学生。

升级方案：

- 结构公式放最上方。
- 中文解释短一些。
- English / Русский 放在支架区。
- 例句与结构对应，逐条展示。
- 例句可以做“结构高亮”，例如：

```text
我 / 是 / 学生
Subject / Verb / Object
```

### 3. 课文模块

当前已处理：

- 增加英文模式。
- 拼音补声调。

升级方案：

- 四种模式作为分段按钮：
  - 拼音。
  - 汉字 + 拼音。
  - 汉字 + 拼音 + 英文。
  - 英文。
- 当前句高亮，其他句子弱化。
- 朗读按钮只出现在当前句，减少视觉噪音。

### 4. 练习模块

当前重点：

- 选词填空。
- 连词成句。

升级方案：

- 每题只显示一个核心任务。
- 答题后给即时反馈。
- 连词成句使用 `acceptable_answers` 判断，不只依赖唯一答案。
- 如果学生端扫码答题接入，反馈逻辑必须与教师端一致。

### 5. PK 模块

当前已确认：

- PK 英文题面可以保留。

升级方案：

- PK 题目保持短、中英双语。
- 队伍分数大而清楚。
- 答案揭示要有明显状态。
- 不在 PK 中加入过多复杂赛制，先保留高激活度。

## 六、推荐文件规划

短期不重构也可以，但建议先定目标结构：

```text
source/in-class/
  teacher.html
  data_L02.js
  styles/
    classpro-theme.css
    classpro-layout.css
  modules/
    vocab-card.js
    grammar-teach.js
    text-reader.js
    vocab-fill.js
    sentence-ordering.js
    pk-quiz.js
  core/
    state.js
    scoring.js
    realtime.js
```

第一轮轻量升级可以只做：

```text
source/in-class/styles/classpro-lite.css
```

并先在 `teacher.html` 中引入，减少一次性拆 JS 的风险。

## 七、实施顺序

### Step 1：视觉变量和布局先统一

- 提取颜色、间距、圆角、字号。
- 优化顶部栏、主舞台、底部控制。
- 保持功能不变。

验收：

- 页面看起来不像旧 ClassPro 直接搬过来。
- 教师不会找不到原有功能。

### Step 2：词汇卡片升级

- 替换配图。
- 默认隐藏扩展例句。
- 减少词汇卡片信息密度。

验收：

- 一屏能看清一个词。
- 学生能通过图片先猜意思。

### Step 3：语法和课文阅读升级

- 语法结构更清楚。
- 例句与结构对应。
- 课文四种显示模式稳定。

验收：

- 俄语学生能借助 English / Русский 理解结构。
- 切换模式不触发无关工具。

### Step 4：练习与 PK 统一反馈样式

- 正确、错误、参考答案、待教师确认统一视觉。
- PK 和普通练习不使用完全不同的反馈语言。

验收：

- 学生知道自己做对了没有。
- 教师知道什么时候需要现场判断。

### Step 5：再接入第一批新题型

在界面稳定后接入：

- 图片猜词。
- 完成句子。
- 课文判断。

## 八、本周建议只做的范围

建议本周只做：

- 配图规范确认。
- ClassPro+ 轻量视觉方案确认。
- `teacher.html` 小幅样式升级。
- 不做大规模 JS 模块拆分。
- 不做全部题型接入。

这样可以守住 HSK1-L02 样课闭环，不把工作量炸开。

## 九、验收标准

本阶段完成后，应满足：

- 教师能明确区分 ClassReady+ 和 ClassPro+。
- ClassPro+ 页面有明显升级感。
- 词汇图片有统一生产规范。
- HSK1 前几课默认低认知负荷。
- 已有 6 个基础模块功能不回退。
- 后续接入图片猜词、完成句子、课文判断时不需要再重想界面结构。
